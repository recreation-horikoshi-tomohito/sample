# Feature Specification: 社員一覧

**Feature Branch**: `001-employee-list`
**Created**: 2026-04-20
**Status**: Draft
**Input**: 社員一覧画面を作りたい。この機能ではどんな社員がいるか、それぞれの役割や役職、年齢、勤続年数など多くの情報がわかります

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 在籍中の社員一覧の取得 (Priority: P1)

管理者・担当者が在籍中の社員の情報（氏名、役職、部署、年齢、勤続年数など）を一覧で確認できる。退職済の社員は含まれない。

**Why this priority**: 社員管理の基本機能であり、他の機能の前提となる。現在在籍中の社員の全体像を把握するための最重要機能。

**Independent Test**: 在籍中・退職済の社員が混在する状態で一覧取得を実行し、在籍中の社員のみの氏名・役職・部署・年齢・勤続年数が返されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 在籍中の社員が1名以上いる状態で、**When** 社員一覧を取得すると、**Then** 在籍中の全社員の氏名・役職・部署・年齢・勤続年数が返される
2. **Given** 在籍中と退職済の社員が混在する状態で、**When** 社員一覧を取得すると、**Then** 在籍中の社員のみが返され、退職済の社員は含まれない
3. **Given** 在籍中の社員が1名もいない状態で、**When** 社員一覧を取得すると、**Then** 空のリストが返される

---

### User Story 2 - 勤続年数の自動計算 (Priority: P2)

勤続年数は入社日から現在日付を元に自動計算されて返される。手動で更新する必要がない。

**Why this priority**: 勤続年数を手動管理すると毎年更新が必要になり運用コストが高い。自動計算により常に正確な値が得られる。

**Independent Test**: 入社日が異なる複数の社員に対して一覧取得を実行し、各社員の勤続年数が入社日から現在日付までの年数として正しく計算されていることを確認できる。

**Acceptance Scenarios**:

1. **Given** 入社日が5年前の社員がいる状態で、**When** 社員一覧を取得すると、**Then** その社員の勤続年数が5年として返される
2. **Given** 入社日が今年の社員がいる状態で、**When** 社員一覧を取得すると、**Then** その社員の勤続年数が0年として返される

---

### Edge Cases

- 在籍中の社員が1名もいない場合は空のリストを返す
- 勤続年数は年単位（端数切り捨て）で計算する
- 退職済の社員は一覧に含まれない（退職済社員の取得は別APIで対応）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは在籍中の社員のみを一覧形式で返さなければならない（MUST）
- **FR-002**: 各社員の情報には氏名・役職・部署・年齢・勤続年数を含まなければならない（MUST）
- **FR-003**: 勤続年数は入社日から現在日付を元に自動計算されなければならない（MUST）
- **FR-004**: 在籍中の社員が存在しない場合は空のリストを返さなければならない（MUST）
- **FR-005**: 一覧はJSON形式で返さなければならない（MUST）

### Key Entities *(include if feature involves data)*

- **社員（Employee）**: 社員を表すエンティティ。氏名、役職、部署、年齢、入社日、在籍状況を持つ
  - `id`: 社員を一意に識別する識別子
  - `name`: 氏名
  - `role`: 役割（例: エンジニア、デザイナー、マネージャー）
  - `position`: 役職（例: 一般、主任、課長、部長）
  - `department`: 部署
  - `age`: 年齢
  - `hire_date`: 入社日
  - `status`: 在籍状況（在籍中 / 退職済）—フィルタリング用。一覧レスポンスには含まない
  - `years_of_service`: 勤続年数（入社日から現在日付を元に自動計算、レスポンス時のみ）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 社員一覧の取得が1秒以内に完了する
- **SC-002**: 在籍中の全社員の情報が漏れなく返される（取得件数 = 在籍中社員の登録件数）
- **SC-003**: 勤続年数の計算誤差がゼロである（年単位で正確に算出）
- **SC-004**: 在籍中の社員が0名の場合も正常に空リストが返される（エラーなし）
- **SC-005**: 退職済の社員が一覧に含まれない

## Assumptions

- 社員データはシステム内に事前に登録されている（登録機能は本スコープ外）
- 社員エンティティには在籍状況（在籍中/退職済）フィールドが存在する
- 退職済社員の取得は別エンドポイントで対応予定（本スコープ外）
- 一覧の並び順は特に指定なし（登録順）
- ページネーション・検索・フィルタリングは本スコープ外（シンプルな全件取得）
- 年齢は社員エンティティに直接保持する（誕生日からの自動計算は対象外）
- 認証・認可は本スコープ外
